# 模式 C：课件精炼 — 详细模板

## 适用场景

有 PPT 课件或幻灯片风格的要点罗列（bullet points 密集、缺少过渡和逻辑连接），需要补全为连贯可读的笔记。

## 核心原则

1. **保留所有关键信息**：原始课件每个要点都要体现
2. **补充逻辑连接**：用"因此""例如""具体来说""需要注意的是"串联
3. **按主题重组**：如果课件顺序混乱，按知识逻辑重新排列
4. **保持简洁**：精炼不是扩写，而是让原文更易读

## 输出结构

### 主题段落

每个核心主题形成一个段落式笔记：

```markdown
## [主题名称]

**[一句话主题概述]**

原始课件提到以下几个要点：A、B、C。

**A 的详细说明**：[将要点 A 展开为 1~3 句话]
- A 的前提是：[补充逻辑关系]
- A 的推论是：[引出 B]

**B 的详细说明**：[将要点 B 展开为 1~3 句话]
- B 与 A 的[关系]

**C 的详细说明**：[将要点 C 展开为 1~3 句话]

**需要注意**：
- [易错点或特殊情况]
```

### 原始要点对照

每个主题下用 blockquote 保留原始课件要点：

```markdown
> **原始课件要点**：
> - 要点1
> - 要点2
```

### 整体逻辑

```markdown
## 整体逻辑

[一段话概括知识脉络]
```

## 精炼示例

**原始输入（课件风格）**：
```
二分查找
- 前提：有序数组
- 每次比较中间元素
- 时间复杂度 O(log n)
- 适用：静态查找
- 不适用：频繁插入删除
```

**精炼输出**：
```
## 二分查找

**二分查找**是一种在有序数组中快速定位目标的算法。每次取中间元素比较，每次减少一半范围，时间复杂度 O(log n)，远优于顺序查找的 O(n)。

需要注意的是，二分查找仅适用于静态查找场景（数据不频繁变化）。

> **原始课件要点**：
> - 前提：有序数组
> - 每次比较中间元素
> - 时间复杂度 O(log n)
> - 适用：静态查找
> - 不适用：频繁插入删除
```

## 完整合并结构

```
# 课程名称 - 精炼笔记

## [核心主题1]
> 原始课件要点
> [精炼段落]

## [核心主题2]
> 原始课件要点
> [精炼段落]

## 整体逻辑
```
